Описание проекта
В данном проекте представлено решение 5 аналитических задач на языке SQL. Задачи направлены на анализ данных о клиентах, их платежах и географической принадлежности. Работа выполнялась в среде PostgreSQL.

Структура данных
Для анализа использовались три таблицы:

payments:

id_client — уникальный идентификатор клиента.

time_payment — дата и время платежа.

amt_payment — размер платежа.

client_info:

id_client — уникальный идентификатор клиента.

gender — пол клиента.

age — возраст клиента.

id_city — идентификатор города клиента.

city_info:

id_city — идентификатор города клиента.

name_city — название города.

name_region — наименование федерального округа.

Решаемые задачи и SQL-запросы
Задание 2.1: Динамика платежей в Поволжье
Цель: Вывести динамику суммарных платежей по дням для федерального округа "Поволжье".

Задание 2.2: Доля мужчин по городам
Цель: Для каждого города найти долю мужчин (% от всех клиентов) в возрастной категории от 20 до 40 лет.

Задание 2.3: Средний возраст неактивных клиентов
Цель: Определить средний возраст клиентов, которые ни разу не совершали платежей.

Задание 2.4: Первые три платежа по округам
Цель: Для каждого федерального округа выделить три самых первых платежа.

Задание 2.5: Среднее время между платежами
Цель: Для городов в округах "Южный" и "Северный" рассчитать среднее время между платежами одного клиента. Результат представлен в удобочитаемом формате (дни, часы, минуты, секунды).

Технологии и навыки
Язык: SQL (PostgreSQL)

Основные конструкции:

JOIN для объединения таблиц

Группировка и агрегатные функции (GROUP BY, SUM, AVG, COUNT)

Оконные функции (ROW_NUMBER, LAG)

Условные выражения (CASE)

Работа с датами и временем (EXTRACT, EPOCH, преобразование типов)

Общие табличные выражения (CTE)

Примечание: Для проверки работоспособности запросов необходимо предварительно развернуть тестовые таблицы с данными.
